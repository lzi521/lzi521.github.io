前端如何实现token的无感刷新？

第一种，由后端同学来返回过期时间，前端在每次请求的时候，判断token的过期时间，如果快到过期时间，就去调用刷新token接口。不过，这种方案不仅需要后端额外提供一个token过期时间，而且当前端用本地时间判断，本地时间又被修改的情况下，就会造成判断不准确，所以不推荐。

第二种，前端写个setInterval定时器，定时去检查距离用户登录时间是否超过某个时间段，然后去刷新token接口，这种方案比较浪费资源，所以不建议使用。

第三种，可以在axios请求响应拦截器中拦截，当判断token过期后，调用刷新token的接口，这种方案不需要后端额外返回字段，也不会浪费资源，是比较推荐的，不过，这种方案实现的时候，需要注意几点，

第一点就是需要防止多次刷新token。

强缓存和协商缓存同时存在的情况：如果强缓存仍在有效期内，则强缓存覆盖协商缓存，即优先使用本地缓存资源；若强缓存过期，则协商缓存生效。
强缓存字段expires和cache-control同时存在的情况：cache-control会覆盖expires，无论expires指示的资源是否过期，都以cache-control为准。
协商缓存字段Etag和Last-Modified同时存在的情况：Etag会覆盖Last-Modified，即优先根据Etag判断资源是否更新。

es6的新特性

ES6引入了许多新特性，包括箭头函数、模板字符串、解构赋值、Promise、let与const声明变量、for…of循环、Classes和模块化。ES6（ECMAScript 2015）是JavaScript语言的一个重要更新，它为Web开发带来了一系列现代的、强大的特性，极大地提高了编码效率和代码的可维护性。以下是简要介绍：

  1. 箭头函数：箭头函数提供了一种更简洁的函数语法，它的一个显著特点是不绑定自己的this值，而是继承外层函数的this值。这在处理回调函数时尤其有用，因为它消除了管理this值的需要[^1^][^3^]。
  2. 模板字符串：模板字符串允许嵌入表达式，并支持多行字符串，使得字符串的处理更加灵活和强大。这对于构建动态字符串非常有用，特别是在需要格式化输出时[^1^][^3^]。
  3. 解构赋值：解构赋值允许从数组或对象中方便地提取值，并将其分配给新的变量，这大大简化了数据操作和交换过程。这使得代码更加清晰和简洁[^1^][^3^]。
  4. Promise：Promise 提供了一种更优雅的方式处理异步操作，通过链式调用和错误处理，替代了传统的回调函数模式。Promise使得异步代码更容易理解和维护[^1^][^3^]。
  5. let与const声明变量：引入了新的变量声明方式，let和const。let用于声明块级作用域的变量，而const用于声明常量，其值一旦设置便不可更改。这些新关键字有助于提高代码的清晰性和避免潜在的错误[^1^][^3^]。
  6. for…of循环：这是一种新的迭代器，专门用于遍历如数组、Map、Set等可迭代对象的值。这与之前用于遍历对象键的for...in循环形成对比，提供了更直观的迭代方式[^1^][^3^]。
  7. Classes：ES6引入了类的概念，尽管它们在底层仍然基于原型继承，但类的语法让JavaScript的对象导向编程更加接近传统OOP语言的风格。类语法促进了代码的组织和复用[^1^][^3^]。
  8. 模块化：ES6提供了原生的模块系统，通过import和export关键字来导入和导出模块成员。这解决了以往依赖第三方库如RequireJS的问题，使得模块管理更为标准化和简单[^1^][^3^]。

此外，除了上述特性，还有Symbol类型的引入，它允许创建独一无二的标识符，以及扩展运算符(...)，可用于函数参数展开或数组和对象的合并等多个场景。这些新特性共同提升了JavaScript的表达能力和开发效率[^3^]。

综上所述，ES6的新特性极大地丰富了JavaScript的表达能力，提高了代码的效率和可维护性。掌握这些新特性对于任何使用JavaScript开发的工程师来说都是必要的，它们不仅可以帮助编写更加清晰和高效的代码，还能确保开发者能够充分利用JavaScript语言的最新进展。

Promise和async，await的区别

在JavaScript中处理异步操作的不同方式，Promise、async/await各自提供了不同的抽象层次和方法来简化异步代码的编写和管理。以下是具体分析：

  1. 基本概念与用法

    • Promise：Promise 是一种编程模型，它代表一个最终可能完成（也可能被拒绝）的操作，提供了一个更合理的异步模式[^1^]。Promise有三种状态：pending（挂起）、fulfilled（已实现）、rejected（已拒绝）。一旦状态改变（从pending变为fulfilled或rejected），则不可再变。Promise通过then方法来处理完成或失败后的逻辑，通过catch方法来捕获错误。
    • Async/Await：Async/Await构建在Promise之上，是对其的进一步封装和改进。async关键字用于声明一个函数是异步的，而await关键字用于等待一个Promise解析完成，并返回其结果。Async/Await使得异步代码可以以同步的方式书写，提高了代码的可读性和可维护性[^2^]。

  2. 错误处理

    • Promise：在Promise中，错误处理主要通过.then(...).catch(...)链来实现。如果Promise被rejected，可以在.catch方法中捕获并处理错误。若未设置.catch，则错误会向上冒泡[^4^]。
    • Async/Await：对于Async/Await，任何在async函数内部抛出的错误都可以通过外围的try-catch语句来捕获处理。这使得错误处理更加集中和清晰[^2^]。

  3. 性能与执行上下文

    • Promise：创建Promise时，其后的代码会继续同步执行。当Promise状态改变时，相应的回调函数会被添加到微任务队列中，等待当前执行栈清空后执行[^2^]。
    • Async/Await：在async函数中，await关键字之前的代码仍同步执行。一旦遇到await，JavaScript引擎会暂停该函数的执行，直到Promise解析完毕。这意味着async函数在等待时不会阻塞后续代码的执行，但是函数内部的执行顺序是按书写顺序来的[^2^]。

  4. 使用场景与适用性

    • Promise：适用于简单的异步流程控制，尤其是在只需要处理成功或失败单次逻辑的场景。
    • Async/Await：适合处理更复杂的异步流程，特别是那些涉及多个依赖步骤的情况，能显著提高代码的整洁度和可理解性。

综上所述，虽然Promise和async/await都是处理异步操作的工具，但它们在使用上有明显的不同。async/await通常被认为是更现代化、更易于理解和编码的方法，特别是在处理复杂或者有多个步骤的异步操作时。然而，理解Promise仍然是基础，因为async/await是基于Promise工作的。

async，await的原理

async 和 await 是 JavaScript 中用于处理异步操作的关键词，它们是 ES2017（也称为 ES8）标准引入的特性，旨在提供一种更加简洁和直观的方式来编写异步代码，相比于传统的回调函数和 Promise，它们使得代码结构更接近于同步代码的阅读体验。

async 原理
自动返回 Promise: 当一个函数被 async 关键字修饰时，该函数会自动返回一个 Promise 对象。这意味着无论函数内部如何执行，外部调用该函数时都可以通过 .then() 或者 .catch() 处理这个 Promise。
内置执行器: async 函数有一个内置的执行器，这意味着你不需要像 Generator 函数那样手动调用一个外部函数（如 co 函数库）来执行它。这使得异步代码的书写和调用变得简单直接。

await 原理
暂停执行: await 关键字只能在 async 函数内部使用。当遇到 await 表达式时，async 函数的执行会被暂停，直到等待的 Promise 解析完成（无论是 resolved 还是 rejected）。
处理 Promise: await 后面通常跟着一个 Promise（或者任何可以被转换为 Promise 的表达式）。如果这个 Promise 成功（resolved），那么 await 将返回 Promise 的解析值；如果 Promise 被拒绝（rejected），则 await 会抛出一个错误，这可以在 async 函数中通过 try…catch 语句捕获。
非阻塞: 虽然 await 会使 async 函数在该点暂停，但它并不会阻塞其他代码的执行。JavaScript 的单线程特性意味着在 await 暂停期间，其他任务（比如事件处理）仍然可以继续执行。

实现细节
在底层实现上，async 函数通过将函数体转换为一个状态机来工作，这个状态机能够根据 await 表达式的解析结果恢复执行。每次 await 遇到一个待解决的 Promise 时，状态机会挂起当前的执行上下文，并在 Promise 解析时通过微任务（microtask）队列恢复执行，从而实现了非阻塞的异步控制流。

总之，async/await 提供了一种优雅的方式来管理异步操作，让开发者能够以类似同步代码的方式编写异步逻辑，而无需显式地处理回调函数或 Promise 链。

Promise的原理

Promise 是 JavaScript 中用于处理异步操作的一种模式，它的设计目标主要是为了解决回调地狱（Callback Hell）问题，让异步代码更加易于理解和维护。Promise 的核心原理可以概括为以下几个方面：

状态机模型：Promise 是一个具有三个状态的对象，这三个状态分别是：三种状态不可逆

Pending（等待中）：这是 Promise 的初始状态，表示异步操作正在进行中，既没有成功也没有失败。
Fulfilled（已成功）：表示异步操作已经成功完成，此时 Promise 的结果值可用。
Rejected（已失败）：表示异步操作失败，此时 Promise 会携带一个错误原因。
不可变性：一旦 Promise 的状态从 Pending 变为 Fulfilled 或 Rejected，这个状态就会固定下来，不会再改变。这意味着你不能从 Fulfilled 状态回到 Pending 或 Rejected，也不能从 Rejected 变回 Pending 或 Fulfilled。

链式调用：Promise 提供了 then、catch 和 finally 方法，允许以链式的方式处理异步操作的成功或失败结果。then 方法用于注册在 Promise 成功时的回调函数，catch 用于处理错误，finally 则不管 Promise 是成功还是失败都会执行的代码。

解决回调地狱：通过将回调函数注册到 Promise 上，而非直接嵌套，Promise 可以帮助开发者避免复杂的回调函数嵌套，使异步代码更加扁平和易于阅读。

Promise 规范：Promise 的行为遵循 Promise/A+ 规范，这是一个社区广泛接受的标准，确保了不同实现之间的兼容性。规范中定义了如如何传递结果、错误处理、Promise 的决议过程等细节。

内部实现：Promise 的内部通常包含一个状态机来管理状态变化，以及一个回调函数队列来存放待处理的 onFulfilled 和 onRejected 函数。当调用 resolve 或 reject 方法时，状态会改变，并且根据状态触发相应的回调函数执行。

宏任务与微任务：Promise 的解决（resolve/reject）操作通常会安排在当前执行栈结束后的一个微任务（microtask）队列中执行，这意味着 Promise 的回调函数会在当前同步代码执行完毕后立即执行，但不会阻塞其他宏任务（如事件循环中的其他任务）。

综上所述，Promise 的原理围绕着状态管理、链式调用和异步控制流优化，提供了一种更加优雅的异步编程模型。
